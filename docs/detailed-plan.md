# Детальный план работ (Action Plan)

План составлен на основе ролей (см. `project-roles.md`) и дорожной карты MVP (см. `roadmap.md`).

## 1. Бобровских Егор (Тимлид, Архитектор, UX)

**Цель:** Обеспечить технический фундамент и спроектировать удобный интерфейс.

| Задача | Описание | Срок (Дедлайн) | Статус |
|--------|----------|----------------|--------|
| **Инициализация проекта** | Создание репозитория, настройка линтеров (ESLint, Prettier), выбор структуры папок. | Как карта ляжет | ⏳ |
| **UX-прототипирование** | Создание Wireframes для: Главной, Страницы статьи, Каталога тем. Проработка User Flow. | Как карта ляжет | ⏳ |
| **Архитектура БД** | Проектирование схемы данных (Статьи, Теги, Языки, Комментарии). Выбор ORM. | Как карта ляжет | ⏳ |
| **Code Review** | Проверка кода бэкенда и фронтенда, слияние веток в `main`. | Еженедельно | ♾️ |
| **Координация** | Проведение синков (созвонов) команды. | Еженедельно | ♾️ |

### Общий график (Sprints)

1. **Неделя 1:** Дизайн (UX/UI), Настройка окружения, Сбор материалов.
2. **Неделя 2:** Разработка ядра (Бэкенд + Базовая верстка), Написание текстов статей.
3. **Неделя 3:** Интеграция контента, Реализация фичи "Сравнение языков", Тестирование.
4. **Неделя 4:** Баг-фиксинг, Финальная полировка, Подготовка презентации.

Рассматривается идея пятилетки за 3 дня как оптимальная альтернатива этому плану
